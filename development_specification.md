# K-Indicator (한국 국가지표 AI 자동화 분석 시스템) 개발 명세서

## 1. 프로젝트 개요
한국의 주요 국가 경제지표(기준금리, 소비자물가상승률, 무역수지 등) 발표 시, 공식 부처(기재부, 통계청, 한국은행 등)의 보도자료 원본을 실시간으로 자동 수집하고, AI를 통해 주요 수치와 인사이트를 추출하여 시각화하는 로컬 자동화 대시보드입니다. 뉴스 기사 대신 원본 문서(PDF, HWPX)를 직접 다루어 신뢰도 높은 데이터를 제공합니다.

## 2. 기술 스택 (Tech Stack)

### **Backend**
- **Language**: Python 3.10+
- **Framework**: FastAPI (가벼운 비동기 REST API 서버 및 정적 파일 제공)
- **Database**: SQLite3 (경량 로컬 RDBMS, 수집된 보도자료 및 추출된 지표 데이터 저장)

### **Data Pipeline & Scrapers**
- **Network**: `requests`, `urllib`
- **Parsing**: `beautifulsoup4` (HTML 구조 분석), `feedparser` (정부 브리핑 RSS 파싱)
- **Data Collectors**:
  - `collector.py`: korea.kr(정책브리핑) 및 한국은행 보도자료 RSS/게시판 크롤링
  - `base_rate_scraper.py`: 한국은행 기준금리 과거 이력 웹 크롤링
  - `cpi_scraper.py` & `trade_scraper.py`: 한국은행 ECOS API를 활용한 물가상승률, 무역수지, 수출금액지수 과거 장기 시계열 데이터 수집 (Pagination 기법)
- **Document Extractors**:
  - `pypdf`: PDF 문서 텍스트 추출
  - `zipfile` & `xml.etree.ElementTree`: HWPX, DOCX 파일 등 OCF(Open Container Format) 기반 문서의 내부 XML 구조를 네이티브로 파싱하여 텍스트 추출

### **AI & NLP**
- **Model**: Google Gemini API (`gemini-1.5-flash`)
- **Integration**: `google-generativeai` 라이브러리를 활용하여 프롬프트 엔지니어링 기반 핵심 요약, 시장 영향력 평가, JSON 형태의 지표 수치 구조화 수행

### **Frontend**
- **Core**: HTML5, CSS3, Vanilla JavaScript (ES6+)
- **Visualization**: Chart.js (시계열 트렌드 및 이중축 차트, 동적 색상 매핑)
- **Design Concept**: Glassmorphism (반투명 패널, 배경 블러)

## 3. 주요 UI 형태 및 디자인 (UI/UX)

1. **디자인 테마 (Glassmorphism)**
   - 최신 웹 디자인 트렌드인 글래스모피즘(Glassmorphism)을 적용하여, 백그라운드 그라데이션 위에 반투명한 패널과 `backdrop-filter: blur`를 배치해 세련되고 미래지향적인 룩앤필 제공.
2. **종합 대시보드 뷰**
   - **KPI 카드**: 상단에 가장 중요한 4개 지표(기준금리, 물가상승률, 무역수지, 수출증가율)의 최신 수치를 눈에 띄게 배치.
   - **인터랙티브 차트**: 
     - *기준금리 vs 소비자물가상승률*: 꺾은선 형태의 이중축(Dual-axis) 차트로 상관관계 가시화. 데이터 누락 시 선이 끊기지 않도록 `spanGaps` 속성 적용.
     - *수출증가율 vs 무역수지*: 수치(흑자/적자)에 따라 막대 그래프의 색상이 동적으로 변환되는 혼합 차트 구성.
3. **데이터 테이블 및 상세 모달(Modal)**
   - **실시간 필터링**: 테이블 상단 검색창 및 기관별 필터 버튼을 통한 즉각적인 데이터 탐색.
   - **상세 분석 뷰(Modal)**: 특정 행을 클릭하면 모달창이 부드럽게 나타나며, AI 요약, 추출된 지표, 원본 텍스트, 다운로드 링크 등을 탭(Tab) 형태로 전환하며 볼 수 있도록 UX 설계.

## 4. 핵심 트러블슈팅 및 극복 과정 (Troubleshooting)

### **Q1. 공공기관 특유의 HWPX 파일 내 텍스트 추출 문제**
- **문제**: 정부 부처 보도자료의 상당수가 HWPX 형식으로 배포되는데, 이를 파싱할 수 있는 안정적인 파이썬 라이브러리가 부족함.
- **해결**: HWPX 포맷이 사실상 ZIP 압축된 XML 파일 묶음(OCF)이라는 점에 착안. `zipfile` 모듈로 압축을 해제하고 `xml.etree.ElementTree`를 사용해 내부 `Contents/section0.xml` 등의 파일에서 텍스트 노드(`<hp:t>`)만 순회하며 문자열을 추출하는 네이티브 파서를 직접 구현하여 의존성 없이 빠르고 안정적인 추출 성공.

### **Q2. 비정형 보도자료 문서 내에서 정확한 수치 데이터(JSON) 구조화**
- **문제**: 정규표현식(Regex)만으로는 기관마다, 월마다 조금씩 달라지는 서식(표, 줄바꿈 등)에서 정확히 지표 데이터(예: 3.5%)를 뽑아내는 데 한계가 잦음.
- **해결**: Gemini 1.5 Flash 모델에 "반드시 지정된 JSON 스키마 형식으로만 응답할 것"을 강제하는 프롬프트 엔지니어링을 적용. JSON.parse 오류 방지를 위해 마크다운 코드 블록 찌꺼기 등을 걷어내는 후처리 로직(Sanitization)을 추가. 또한, API 키 미설정 시 작동할 Regex 기반의 Fallback 로직을 이중으로 구성하여 서비스 안정성 보장.

### **Q3. 공공기관 웹사이트의 다양한 인코딩 혼재 (EUC-KR vs UTF-8)**
- **문제**: 오래된 게시판이나 특정 크롤링 엔드포인트에서 `utf-8` 디코딩 에러가 간헐적으로 발생함.
- **해결**: Response 텍스트 디코딩 시 인코딩을 명시적으로 체크하고, 실패 시 `euc-kr` 혹은 `cp949`로 순차적 디코딩을 시도하는 안전한 디코딩 유틸리티 함수를 래핑하여 크롤링 중단 현상을 방지.

### **Q4. 한국은행 ECOS 과거 시계열 데이터 수집 시 API 키 인증 우회**
- **문제**: 무역수지, 물가상승률 등 과거 수십 년 치의 데이터를 얻으려면 한국은행 ECOS OpenAPI 키 발급이 필수적이나, 사용자의 설치 편의성(Zero-config)을 위해 키 없이 해결해야 함.
- **해결**: ECOS에서 인증키 없이 제공하는 '샘플 API' 기능(최대 10건 반환 한도)을 분석하여, 요청 1회당 10건씩 조회하도록 시작 인덱스를 조정하는 **Pagination (페이지네이션)** 순회 로직을 구현. 이를 통해 수백 건의 과거 데이터를 합법적으로 긁어와 데이터베이스에 성공적으로 시딩(Seeding)함.

### **Q5. ECOS OpenAPI 과도한 호출에 따른 Rate Limit (Error 602) 대응**
- **문제**: Pagination을 이용해 다수의 과거 지표(물가, 무역수지, 주택가격, 실업률 등)를 단시간에 연속 조회할 경우, ECOS 서버에서 `3분간 300회 초과`를 감지하여 해당 IP를 30분간 차단(Error 602)시키는 이슈 발생.
- **해결**: 스크래퍼 내부에 API 호출 간 `time.sleep()` 딜레이를 추가하고, 만약 API 응답이 차단 상태(602)로 반환될 경우 비정상적인 데이터를 DB에 넣지 않고 즉시 스킵(Skip)하도록 안전장치(Graceful degradation) 구현. 차단이 풀린 후 스크립트를 재실행하면 정상적으로 수집되도록 안내 처리함.

### **Q6. 브라우저 보안 정책에 의한 로컬 파일 다운로드(file:///) 차단 문제**
- **문제**: 프론트엔드(`index.html`)에서 수집된 첨부파일(`PDF`, `HWPX`) 원본을 열기 위해 `file:///` 스키마를 <a> 태그 링크로 제공했으나, 최신 브라우저의 CORS 및 보안 정책으로 인해 웹 환경(`http://127.0.0.1`)에서 로컬 리소스 접근이 원천 차단되어 다운로드가 불가능함.
- **해결**: `backend/server.py`에 로컬 파일을 안전하게 스트리밍 방식으로 내려주는 다운로드 전용 API 라우터(`@app.get("/api/download")`)를 신설. 프론트엔드에서는 `file:///` 대신 서버 다운로드 엔드포인트에 `path` 파라미터를 넘겨받아 브라우저 보안 제약을 회피하고 정상 다운로드가 가능하게 수정.

### **Q7. 수출증가율 등 마이너스 성장 구간에 대한 시각적 하이라이트 (Chart.js 고급 스타일링)**
- **문제**: 시계열 차트에서 특정 데이터(예: 수출증가율)가 0% 미만으로 떨어지는 구간을 사용자가 직관적으로 인지하기 어려움.
- **해결**: Chart.js 3+ 버전의 `fill: { target: {value: 0}, below: 'color' }` 속성과 `segment` 플러그인 속성을 활용. 데이터 값이 0 미만일 때 기준선 아래 영역을 붉은색 반투명으로 칠하고 꺾은선 색상도 붉은색으로 동적 변경되도록 설정. 추가로 자바스크립트 배열 순회를 통해 연속된 마이너스 기간(예: '2023-01 ~ 2023-11')을 계산하여 차트 상단에 명시적 경고 알림 박스로 렌더링되게 구성.
